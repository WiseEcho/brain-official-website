# 项目经验教训

## 2026-06-30：设计系统必须先审计现有风格

**问题**：
用户在看到设计系统演示后反馈「样式和原来差别较大，要严格遵守原来的风格样式」。

**原因**：
我在做设计规范调研和 demo 时，过度依赖通用 SaaS 设计趋势（卡片阴影、圆角、简洁白底），没有先充分审计现有官网页面的真实视觉语言（颜色使用方式、间距、组件形态、动效、图文混排节奏）。

**规则**：
- 做任何设计系统/样式重构前，必须先完整阅读现有页面 HTML 和 CSS，提炼出**原有设计特征清单**
- 新增规范应是对现有风格的系统化、补足，而不是另起一套新风格
- demo 页面必须使用现有真实素材和类名，确保用户能直观对比「和原来一致」

**如何应用**：
1. 读取 index.html、products.html、pricing.html、about.html 等关键页面
2. 记录原站使用的：主色/辅色/背景色、标题字号、卡片样式、按钮样式、间距节奏、动效类型
3. 新设计系统文档中明确标注「继承自现有设计」vs「新增规范」
4. demo 页复用现有 main.css 和 Tailwind 类名，只做结构层面的演示
